Unix e Bash - Parte 1

Vamos aprender como funciona o Unix e como interagir com ele através do terminal, utilizando o Bash.

Quais são seus principais componentes?

Basicamente o unix é montado em três níveis

  1. Kernel - é o responsável por fazer a interação entre esses dois.
    1. Hardware - funciona assim que o computador é ligado usando todo hardware indispensável para a maquina (monitor, placa de vídeo, Processador etc..)
    2. Software - são os programas que você utiliza em seu sistema operacional (photoshop, zoom, Visual studio etc..)

  2. Shell - é a inferface entre o sistema operacional, o usuário e o núcleo do sitema
    1. Sistema Operacional - Windows, Linux etc...
    2. Você
    3. Kernel - Cmd, Terminal e ou Interface gráfica

    Tipos de Shell - SH / BASH / CSH / TCSH / ZSH

  3. Aplicativos - Softwares e os comandos do terminal (Processos e Arquivos)
    1. Processos - Tudo oque você faz no Unix é um processo.
      1. PID - Identificador de processos, que carrega algumas iformações tipo.
        1. Usuário criador
        2. Seção do shell
        3. Estado atual do processo, se está funcionando, suspenso ou aguardando
        4. Linha de comando utilizada etc..

    2. Arquivos - no Unix tudo é um arquivo (diretórios, arquivos, dispositivos etc..)
      1. / - Raiz representado pela barra
      2. Boot - sendo o arquivo de comparação para inicialização
      3. Bin - Aplicativos binários
      4. Dev - arquivos do dispositivo
      5. Etc - scripts de inicialização
      6. Home - diretórios home para diferentes usuários
      7. Proc - diretório dinâmico especial que mantem informações sobre o estado do sistema, incluindo os processos atualmente execultados
      8. Root - é o diretório Home do usuário root
      9. Sbin - são os binários de sistemas importantes
      10. Tmp - são os arquivos temporários
      11. Usr - aplicativos e arquivos que são na maioria das vezes disponiveis aos acessos de todos usuários
      12. Var - são os arquivos variáveis como logs e banco de dados

Quais suas variações?

Trabalhando com diretórios

ls - lista todos os arquivos e diretórios que estão em seu diretório atual

Variações

  1. ls -a - lista arquivos ocultos, mostrando tudo.
  2. ls --help 
                    
    -a, --all        - não ignora entradas começando com .
    -A, --almost-all - não lista as entradas implícitas . e ..
    --author         - com -l, emite o autor de cada arquivo
    -b, --escape     - emite escapes no estilo C para caracteres não-gráficos
    --block-size=TAM - com -l, escala os tamanhos por TAM ao exibi-los; ex., '--block-size=M'; veja o formato de TAM abaixo
    -B, --ignore-backups - não lista as entradas implícitas terminadas com ~
    -c               - com -lt: ordena por, e mostra, ctime (hora da última modificação da informação de estado do arquivo); com -l: mostra o ctime e ordena por nome demais casos: ordena por ctime, mais novos primeiro
    -C               - lista as entradas em colunas
    --color[=QUANDO] - controla se usa cores para distinguir os tipos de arquivo. QUANDO pode ser "never" (nunca), "always" (sempre) ou "auto" (automaticamente)
    -d, --directory  - lista os diretórios em si, e não seu conteúdo
    -D, --dired      - gera a saída projetada para modo "dired" do Emacs
    -f               - não ordena, habilita -aU, desabilita -ls --color
    etc...
  3. cd + destino - entra na página de destino
  4. clear - limpa o terminal
  5. mkdir - cria um novo diretório (pasta)
  6. cd . - mostra o local aonde você está
  7. cd .. - volta para o diretório(pasta) anterior
  8. pwd - mostra o diretório completo, desde o root até a sua posição atual

Operações com arquivos e diretórios

Comandos - cp | cat | rm | mv | rmdir

  1. ls - Sempre verificar quais diretórios tem em sua posição atual.

  2. cp - para copiar um arquivo

    1. cp + nome arquivo + nome arquivo cópia
    2. obs: não esqueça de colocar a extenção do arquivo ex: .txt | .html | .js

  3. cat - abre arquivos e mostra seu conteúdo

    1. cat + nome arquivo.txt
    2. mostrará o conteúdo de nome arquivo.txt no terminal

  4. rm - Remove(exclui) arquivos com exteção, ex: .txt, .html, .js, etc...

    1. rm + nome arquivo
    2. obs: não esqueça de colocar a extenção do arquivo ex: .txt | .html | .js

  5. mv - Move arquivos em geral para um outro local

    1. mv + nome arquivo + /local de destino
    2. com isso conseguimos mudar diretórios e arquivos de um lugar para outro

  6. rmdir - Remove(exclui) diretórios(pastas)

    1. rmdir + nome diretório
    2. Só funciona se o diretório(pasta) estiver vazio

  7. rm -rf - Remove diretório com todo o seu conteúdo dentro

    1. rm -rf + nome diretório
    2. -r - indica que quer remover arquivos e diretórios
    3. f - significa que não é preciso nenhum tipo de confirmação

Exibir conteúdos de arquivos no terminal.

Comandos - less | head | tail | grep | wc

less

  1. less + nome arquivo
  2. Abre o arquivo em modo de exibição
  3. o comando less abre o arquivo no limite do terninal, como se fosse um livreto e para mudar de página, basta clicar na 'tecla espaço' no final aparecerá 'END' indicando que chegou ao fim
  4. q - para sair do comando less
  5. /frase - pesquisará 'frase' dentro do arquivo | n - navegará entre as frases encontradas

head

  1. mostra as 10 primeiras linhas do texto
  2. head + nome arquivo
  3. head + -5 + nome arquivo - mostrará as 5 primeiras linhas do texto

tail

  1. mostra as 10 ultimas linhas do texto
  2. tail + nome arquivo
  3. tail + -5 + nome arquivo - mostra as 5 ultimas linhas do texto

grep

  1. Busca por palavras apontadas
  2. grep + frase buscada + nome arquivo - tem que ser palavras exatas
  3. grep + -i + frase buscada + nome do arquivo - mostra a 'frase buscada' tanto maiúscula quanto minúscula
  4. grep + -i + 'frase buscada' + nome do arquivo - mostra a 'frase buscada entre aspas' tanto maiúscula quanto minúscula
  5. grep + -iv + 'frase buscada' + nome arquivo - mostrará toda a lista menos a frase que está entre aspas 'frase buscada'
  6. grep + -in + 'frase buscada' + nome arquivo - mostrará a frase buscada e a posição em que ela se encontra
  7. grep + -ic + 'frase buscada' + nome arquivo - mostrará quantas vezes o arquivo se repete.

wc

  1. wc + nome arquivo
  2. o resultado é semelhante a isso: 29 92 545 nome arquivo
  3. explicação: 1º - a quantidade de linhas | 2º - a quantidade de palavras | 3º a quantidade de letras(caracteres)
  4. wc -w nome arquivo - mostra apenas a quantidade de palavras
  5. wc -l nome arquivo - mostra somente o total de linhas do arquivo
  6. wc - -c nome arquivo - mostra apenas a quantidade de caracteres do arquivo
